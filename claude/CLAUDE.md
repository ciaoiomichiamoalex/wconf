## Code style

- Follow the naming conventions for filenames and variables specific to the language used, but prefer lowercase with underscores (e.g. `python_reviewer.py`).
- Use 4-space indentation for source code or 2-space indentation only for JSON files.
- Keep the source code concise and clean, avoid redundancy by moving code into dedicated functions.
- Prefer solutions that are efficient in terms of both performance and code writing rather than readability.
- Document the source code appropriately and include comments containing examples.
- Write general documentation about the project in a README.md file, explaining its implementation and how it works.
